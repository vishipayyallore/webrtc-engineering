# Prerequisites (for my path)

## Core web skills

I assume I am comfortable with:

- **HTML** — structure for pages that host calls or chat.
- **CSS** — layout for local preview and simple UIs.
- **JavaScript** — callbacks, promises/async, DOM events, and basic modules.

I am not re-teaching those topics in this repo; I reference MDN when I forget details.

## Helpful but not blocking

| Topic | Why it helps |
|--------|----------------|
| **Node.js** | Small signaling servers and Socket.IO-style labs in `06_demos`. |
| **TCP/UDP basics** | ICE, STUN, and firewall behavior make more sense. |
| **HTTPS / localhost** | Camera, mic, and some APIs require secure contexts. |

## How I study weak areas

If networking feels fuzzy, I pair `03_networking` notes with Wireshark only after I understand the vocabulary here. If Node is rusty, I start with `06_demos/03_websocket_signaling` before peer connection demos.
